# Система:

## Auth Service

Защита. Все просто, есть:
* Привелегии (Permissions)
* Роли (Roles)
* Пользователи (данные входа)

Пользователи могут иметь несколько ролей, и также отдельно назначенные привилегии.

Используется JWT токен на RS256

### Future

* Добавить в конфигурацию возможность менять алгоритм шифрования токена
* Добавить в защиту такие методы как: Safe-Codes, 2FA, Hardware Key. 


## API Gateway

Входная точка. Проксируем запрос во внутреню сеть. REST -> GRPC.

Запрос приходит. Из него достаем данные какой метод какой сервис какой пользователь. Делаем проверку токена с помощью PublicKey.
Далее запрос на AuthService смотрим CheckAccess. Пропускаем пользователя во внутренюю систему.
